Patra URL Monitor — Privacy Policy

Effective date: April 30, 2026  |  Last updated: April 30, 2026

1. Introduction

Patra URL Monitor (“the Extension”) is a browser extension developed and published by Patra Corp (“we,” “us,” or “our”). It is a component of the Patra Endpoint Monitor platform, an internal enterprise security tool used exclusively within Patra Corp. The Extension is not sold to or intended for use by the general public. It is deployed only on systems owned and operated by Patra Corp.

This Privacy Policy describes what data the Extension collects, how that data is handled and stored, and with whom it is shared. By using a Patra Corp–managed device on which the Extension is installed, you acknowledge these practices.

2. Data Collection

The Extension collects the following categories of data from the browser:

  • Page URLs — the full URL (including domain, path, and query parameters) of web pages visited in the browser.
  • Page titles — the document title of each visited page.
  • Tab lifecycle events — events indicating when a tab is opened, closed, activated, deactivated, navigated, or changes audio playback state.
  • Browser type — the name of the browser in which the Extension is running (e.g., Chrome, Edge, Brave, Opera, Vivaldi).
  • Tab metadata — numeric tab and window identifiers, active/inactive state, and audible state. These are transient session identifiers, not persistent user identifiers.
  • Timestamps — the date and time at which each event occurs.

The Extension does not collect:

  • Passwords, form inputs, or autofill data
  • File contents, downloads, or upload data
  • Cookies, authentication tokens, or session data
  • Personal identity information such as names, email addresses, or phone numbers
  • Browsing history beyond the active session
  • Data from internal browser pages (chrome://, edge://, about:, devtools://, etc.)
  • Keystroke or mouse input data

3. Data Handling

All data collected by the Extension is transmitted exclusively to a local agent process (the Patra Endpoint Monitor Agent) running on the same machine. Communication occurs over http://127.0.0.1:18599 (localhost only). The Extension does not transmit any data directly to external servers, remote endpoints, the internet, or any third-party service.

The local agent process is responsible for forwarding the data to Patra Corp’s self-hosted Patra Monitor server over an encrypted (HTTPS/TLS) connection. The Extension itself does not control or manage this onward transmission.

Within the Extension, data is held only in short-lived in-memory caches used for deduplication and debouncing of duplicate events. These caches are automatically cleared every 60 seconds and do not survive browser restarts. The Extension does not write data to the browser’s local storage, IndexedDB, cookies, or any other persistent storage mechanism.

4. Purpose of Data Use

The data collected by the Extension is used solely for the following internal Patra Corp purposes:

  • URL filtering and policy enforcement — blocking or redirecting access to websites that violate organizational security or acceptable-use policies.
  • Security monitoring and threat detection — identifying potentially malicious, phishing, or unauthorized websites accessed from company endpoints.
  • Compliance reporting — generating audit logs and reports for internal compliance and governance requirements.
  • Operational analytics — aggregated, non-personally-identifiable analysis of browsing patterns to improve security policies.

The data is never used for advertising, marketing, user profiling for commercial purposes, sale to third parties, credit assessment, or any purpose unrelated to Patra Corp’s internal security operations.

5. Data Storage

The Extension itself does not store any data persistently. All in-memory caches are ephemeral and cleared automatically as described in Section 3.

Data forwarded by the local agent to the Patra Monitor server is stored on Patra Corp’s self-hosted infrastructure. Storage practices include:

  • Encryption at rest — data is stored using industry-standard encryption (AES-256).
  • Encryption in transit — data transmitted from the agent to the server uses TLS/HTTPS.
  • Access controls — server access is restricted to authorized Patra Corp security administrators via role-based access controls and session authentication.
  • Retention period — collected data is retained in accordance with Patra Corp’s internal data retention policies. Default retention is 90 days, after which data is automatically purged. Retention periods may be adjusted by Patra Corp administrators based on organizational requirements.

6. Data Sharing

Data collected by the Extension is not shared with, sold to, transferred to, or disclosed to any third party, including but not limited to:

  • Advertising networks or platforms
  • Data brokers or information resellers
  • Analytics providers
  • Any external organization, entity, or individual outside of Patra Corp

Data is accessible only to authorized Patra Corp administrators who require access for the security and compliance purposes described in Section 4.

Data may be disclosed only in the following limited circumstances:

  • When required by applicable law, regulation, or valid legal process
  • To protect the security and integrity of Patra Corp systems (e.g., incident response)

7. User Rights and Control

The Extension is deployed and managed by Patra Corp as part of its internal endpoint security program. It is installed on company-owned devices and operates under organizational IT policies.

Patra Corp employees may:

  • Contact the Patra Corp IT or Security team to request information about what data has been collected from their assigned endpoint.
  • Request correction or deletion of data in accordance with applicable internal policies and legal requirements.
  • Direct questions about the scope, purpose, or duration of data collection to their manager or the IT Security department.

8. Chrome Web Store User Data Policy Compliance

The Patra URL Monitor extension’s use and transfer of information received from Google APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. Specifically:

  • Data use is limited to providing and improving the Extension’s single stated purpose: URL monitoring and filtering for enterprise endpoint security.
  • Data is not transferred to third parties except as required for the Extension’s core functionality, for legal compliance, or for security purposes (malware and fraud prevention).
  • Data is not used for serving advertisements or for purposes unrelated to the Extension’s core functionality.
  • Human access to raw user data is limited to authorized Patra Corp security administrators for security investigations, legal compliance, or when aggregated and anonymized for internal operational analysis.

9. Changes to This Policy

We may update this Privacy Policy from time to time to reflect changes in our practices or for legal, operational, or regulatory reasons. When we make changes, we will update the “Last updated” date at the top of this page. Continued use of Patra Corp devices on which the Extension is deployed constitutes acceptance of the updated policy.
